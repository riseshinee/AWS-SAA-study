# 컴퓨트 서비스

## VPC (Virtual Private Cloud)

**정의**: AWS 네트워크 생성 및 관리도구이자 클라우드 인프라 관리 도구

### 플레이스먼트 그룹 (Placement Groups)

#### 1. 클러스터 플레이스먼트 그룹 (Cluster Placement Group)
- 근거리에 위치한 단일 AZ에 연관된 인스턴스를 론칭
- 낮은 지연시간, 높은 처리량이 필요한 경우 사용

#### 2. 스프레드 플레이스먼트 그룹 (Spread Placement Group)
- 동시다발적인 실패 상황에 대처
- 인스턴스를 서로 다른 하드웨어에 분산 배치

#### 3. 파티션 플레이스먼트 그룹 (Partition Placement Group)
- 하나의 파티션으로 묶어서 사용할 수 있도록 함
- 대규모 분산 시스템에 적합

## EC2 (Elastic Compute Cloud)

### 인스턴스 중지 및 재시작

| 저장소 유형 | 중지 시 데이터 | 재시작 시 퍼블릭 IP |
|-----------|-------------|-----------------|
| **EBS 볼륨** | 유지됨 | 비지속성 (변경될 수 있음) |
| **인스턴스 스토어** | 유실됨 | - |

**영구적 IP가 필요한 경우**: Elastic IP 주소 할당

### EC2 스토리지 볼륨

#### Elastic Block Store (EBS)
- **특징**:
  - 하나의 인스턴스에 다수의 EBS 볼륨 부착 가능
  - 모든 EBS 볼륨은 스냅샷 생성 방식으로 복제 가능
  - 암호화 가능 (자동 관리 또는 KMS 통해 관리)
  - **지속형** 저장소

- **사용 권장**:
  - 데이터 지속성이 중요한 경우
  - 재부팅 후에도 데이터가 유지되어야 하는 경우
  - AMI에서 생성된 그대로 인스턴스를 사용하는 경우

#### 인스턴스 스토어 (Instance Store)
- **특징**:
  - **비지속형** 저장소
  - 인스턴스 종료 시 데이터 소실
  - 재부팅해도 데이터 유지

- **사용 권장**:
  - 고속 데이터 입출력 및 처리가 중요한 경우
  - 임시 데이터 저장

### EC2 인스턴스 IP 주소

- **프라이빗 IPv4 주소**: 최소 하나 이상 필수
- **퍼블릭 IP**: 선택적 (인터넷 연결 시 필요)
- **Elastic IP**: 비지속성 퍼블릭 IP 주소를 영구적으로 사용하고 싶을 때 할당

### 시큐리티 그룹 (Security Groups)

**기본 동작**:
- 인스턴스로 향하는 모든 **유입 트래픽**: 거부
- 인스턴스에서 나가는 모든 **유출 트래픽**: 허용

### NAT 디바이스 (Network Address Translation)

**목적**: 인터넷 연결 없이 프라이빗 IP 주소를 통해 인터넷에 접속 가능

**옵션**:
- NAT 인스턴스
- NAT 게이트웨이

### 키페어 (Key Pair)

- **퍼블릭 키**: EC2 서버에 저장
- **프라이빗 키**: 로컬 머신에 저장 (절대 공유 금지)

## Auto Scaling

### 오토 스케일링 그룹 (Auto Scaling Group)

**정의**: 오토 스케일링이 관리하는 EC2 인스턴스 그룹

**필수 요소**: 론치 템플릿(Launch Template) 생성

### 용량 설정

| 설정 | 설명 |
|-----|-----|
| **최소용량 (Minimum)** | 인스턴스 수가 이 크기 이하로 내려가지 않음 (0으로 설정 시 모든 인스턴스 종료) |
| **최대용량 (Maximum)** | 인스턴스 수가 이 크기 이상이 되지 않음 (예산 제한선 고려) |
| **희망용량 (Desired)** | 사용자가 희망하는 최소 ~ 최대 사이의 값 |

### 오토 스케일링 옵션

#### 1. 수동 스케일링
- 사용자가 직접 용량 조정

#### 2. 동적 스케일링 정책

##### 단순 스케일링 정책 (Simple Scaling)
- 희망 용량대로 인스턴스를 증가시킴 (타입에 따라 조절)
- 지정 숫자만큼 희망 용량 증가
- 용량을 지정 숫자만큼 증가
- 비율대로 증가

##### 단계별 스케일링 정책 (Step Scaling)
- 메트릭 값에 따라 단계별로 조정

##### 목표 추적 정책 (Target Tracking)
- 특정 메트릭 목표값에 따라 자동 조정

#### 3. 예약된 작업 (Scheduled Actions)
- 정해진 시간에 스케일링 수행

## AWS Systems Manager

**정의**: 리소스 및 온프레미스 서버의 자동 및 수동 작업 관리

### 액션 (Actions)

- **자동화액션**: 자동으로 수행되는 작업
- **명령 액션**: 명령 기반 작업
- **정책 액션**: 정책 기반 작업

### 패치 매니저 (Patch Manager)

**목적**: 인스턴스 패치 업무의 자동화 도구

### 스테이트 매니저 (State Manager)

**목적**: 정의한 방식대로 소프트웨어를 가져오고 환경을 설정하는 환경설정 도구

### Systems Manager Inventory

**목적**: 인스턴스의 데이터를 수집

**수집 항목**:
1. OS 이름 및 버전
2. 어플리케이션 및 파일명, 버전, 사이즈
3. IP 및 MAC 주소 등 네트워크 환경설정
4. Windows 업데이트, 롤, 서비스, 레지스트리 값
5. CPU 모델, 코어, 스피드

## 컨테이너 서비스

### ECS (Elastic Container Service)

**정의**: 컴퓨트 리소스 실행을 스크립트화해 좀 더 효율적으로 처리

### AWS Fargate

**정의**: 컨테이너 서비스들이 안전하게, 격리된 환경에서 실행될 수 있도록 도움
- 서버리스 컨테이너 실행 엔진

### ECR (Elastic Container Registry)

**정의**: 관리형 도커 컨테이너 레지스트리 서비스
- Docker 이미지 저장 및 관리
