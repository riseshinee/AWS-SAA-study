# 핵심 AWS 서비스

## 클라우드 핵심 개념

### 가상화 (Virtualization)
하나의 물리적 서버에 존재하는 하드웨어 리소스를 여러 개의 작은 유닛으로 나누는 기술

### 확장성 (Scalability)
사용자의 애플리케이션에 예측하지 못한 트래픽이 몰렸을 때, 이를 해소하기 위한 리소스를 자동으로 추가하는 능력

### 탄력성 (Elasticity)
요구 수준 변화에 대해 시스템이 어떻게 반응할 것인지 정의하고 관리하는 능력

---

## 컴퓨트 (Compute)

### 1. EC2 (Elastic Compute Cloud)
- CPU, 메모리, 스토리지, 네트워크 인터페이스 프로필 등과 함께 프로비저닝 가능한 가상화된 리소스
- 유연한 인스턴스 타입 선택 및 커스터마이징 지원

### 2. Lambda
- 서버리스 아키텍처 기반 서비스
- 미리 지정한 코드를 실행하는 완전 관리형 서비스
- 인프라 관리 불필요

### 3. Auto Scaling
- 자동화된 컴퓨팅 리소스의 스케일 아웃(Scale-out) 및 스케일인(Scale-in) 도구
- 트래픽 변화에 따른 자동 조정

### 4. Elastic Load Balancer
- 네트워크 트래픽 병목현상이 발생하지 않도록 다수의 서버에 분산시키는 역할
- 고가용성 및 안정성 제공

### 5. ECS (Elastic Container Service)
- 도커 또는 쿠버네티스 같은 컨테이너 기술을 이용해 워크로드 수행
- 컨테이너화된 애플리케이션 관리

### 6. Elastic Beanstalk
- AWS 컴퓨트와 네트워크 인프라의 프로비저닝을 추상화한 관리 서비스
- 개발자는 코드 작성에만 집중 가능

---

## 네트워킹 (Networking)

### 1. VPC (Virtual Private Cloud)
- EC2 및 인스턴스 호스팅을 위해 만들어진 고도의 환경설정이 가능한 네트워크 환경
- 격리되고 커스터마이징 가능한 네트워크 인프라

### 2. Direct Connect
- 서드파티 공급자와 AWS를 연결하여 로컬 데이터 센터 또는 VPC 네트워크 구현
- 전용 네트워크 연결

### 3. Route 53
- DNS(Domain Name System) 서비스
- 도메인 등록 및 라우팅 관리

### 4. CloudFront
- CDN(Content Delivery Network) 서비스
- 글로벌 엣지 로케이션을 통한 콘텐츠 배포 가속화

---

## 스토리지 (Storage)

### 1. S3 (Simple Storage Service)
- 객체 기반 스토리지 서비스
- 확장성 있는 데이터 저장소

### 2. S3 Glacier
- 저렴한 객체 저장 서비스
- 장기 데이터 스토리지 및 백업 용도로 활용
- 아카이브 목적의 스토리지

### 3. EBS (Elastic Block Store)
- 운영체제 및 각종 실행 데이터를 호스팅하기 위한 지속형 가상 스토리지 드라이브
- EC2 인스턴스에 연결되는 블록 스토리지

### 4. Storage Gateway
- 온프레미스 환경과 연결하여 사용할 수 있도록 해주는 하이브리드 스토리지 시스템
- 로컬 데이터 센터와 AWS 클라우드 간의 연결

---

## 데이터베이스 (Database)

### 1. RDS (Relational Database Service)
- 관계형 데이터베이스 관리 서비스
- 자동 백업 및 패치 지원

### 2. DynamoDB
- NoSQL 데이터베이스 서비스
- 고성능 및 확장성 제공

---

## 애플리케이션 관리 (Application Management)

### 1. CloudWatch
- 리소스 및 애플리케이션 모니터링 서비스
- 메트릭, 로그, 알람 관리

### 2. CloudFormation
- 리소스 배포를 위한 템플릿 제공 및 관리
- IaC(Infrastructure as Code) 구현

### 3. CloudTrail
- 사용자 계정의 기록을 수집하여 감사업무 수행 가능
- 규정 준수 및 보안 감시

### 4. Config
- AWS 리소스의 구성 변화 추적 및 규정 준수 확인
- 리소스 인벤토리 관리

---

## 보안 및 권한 관리 (Security & Identity)

### 1. IAM (Identity and Access Management)
- 사용자, 그룹, 역할 관리
- 세분화된 권한 제어

### 2. KMS (Key Management Service)
- 보안 데이터의 암호화 키 생성 및 관리
- 암호화 키 생명주기 관리

### 3. Directory Service
- 사용자 및 컴퓨터 관리를 위한 디렉토리 서비스
- Active Directory 통합

---

## 애플리케이션 통합 (Application Integration)

### 1. SNS (Simple Notification Service)
- 메시지 발행-구독(Pub-Sub) 서비스
- 푸시 알림 및 이메일 발송

### 2. SWF (Simple Workflow)
- 디지털화할 수 없는 이벤트를 포함하는 연속적인 관리 서비스
- 워크플로우 조율 및 상태 관리

### 3. SQS (Simple Queue Service)
- 이벤트 기반 메시징 서비스
- 메시지 큐 관리

### 4. API Gateway
- REST 및 WebSocket API 생성, 관리, 모니터링
- API 보안 및 스케일링

---

## 마이그레이션 서비스

### 1. AWS Migration Hub
- 기존의 워크로드를 AWS 클라우드로 이전하는 과정을 추적하기 위한 서비스
- 마이그레이션 진행 상황 가시화

### 2. AWS Application Migration Service
- 비 클라우드 기반 애플리케이션 서버를 AWS 기반 서버로 이전하기 위한 검증 및 전환 자동화 도구
- 애플리케이션 마이그레이션 자동화
