# AWS VPC (Virtual Private Cloud)

## VPC 기본 개념

### VPC란?
- **정의**: 여러 AWS 서비스에 네트워크 리소스를 담을 수 있는 가상 네트워크
- **범위**: 한 리전 안에서만 존재
- **특징**: AWS 리소스들을 논리적으로 격리된 네트워크 환경에 배치
---

## VPC CIDR 블록

### CIDR란?
- **CIDR (Classless Inter-Domain Routing)**: IP 주소 할당 방법

### CIDR 블록의 역할
- VPC 내의 인스턴스 및 리소스에 할당되는 IP 주소를 결정
- 기본 CIDR 블록은 반드시 할당되어야 함

### CIDR 표기법과 프리픽스
| 프리픽스 | 주소 개수 | 설명 |
|---------|---------|------|
| /16 | 65,536개 | 대규모 네트워크 |
| /24 | 256개 | 중규모 네트워크 |
| /28 | 16개 | 소규모 네트워크 |

- **유효한 IPv4 프리픽스 길이**: /0 ~ /32
- **보조 CIDR 블록**: 지정 가능 (추가 IP 주소 범위)
- **IPv6 CIDR**: 지정 불가능
---

## 서브넷 (Subnet)

### 서브넷의 정의 및 역할
- **정의**: VPC에 있는 네트워크 로직 컨테이너
- **목적**: EC2 인스턴스와 같은 VPC 리소스를 연결
- **기능**: 
  - 네트워크 상의 여러 인스턴스를 격리
  - 인스턴스 간의 트래픽 유입 및 유출 제어
  - 인스턴스는 서브넷 내에서 생성됨

### 서브넷 CIDR 블록
- **예약 IP 주소**: 처음 4개 + 마지막 1개 (총 5개) 예약
  - 첫 번째: 네트워크 주소
  - 두 번째: AWS 라우터
  - 세 번째: AWS DNS
  - 네 번째: AWS 예약
  - 마지막: 브로드캐스트 주소
- **제약사항**: 서브넷은 여러 개의 CIDR을 가질 수 없음

### 서브넷의 가용영역 (AZ)
- 서브넷은 하나의 가용영역 내에서만 존재 가능
- 고가용성 구성 시에는 여러 AZ에 서브넷 배치 필요
---

## 네트워크 인터페이스

### ENI (Elastic Network Interface)
- **역할**: 다른 네트워크 리소스와 통신할 수 있도록 함
- **특징**: 인스턴스와 독립적으로 존재 가능
- **용도**: EC2 인스턴스에 네트워크 연결성 제공

### 성능 강화 네트워크
- 고속의 네트워크 처리속도 제공
- 저지연성 제공
- 고성능 애플리케이션에 필수
---

## 인터넷 게이트웨이 (IGW)

### 정의 및 역할
- 인터넷에서 들어오는 요청을 수신할 수 있도록 함
- VPC 리소스와 인터넷 간의 통신을 중개

### 특징
- AWS 리소스 ID를 식별용으로 할당함
- VPC와 1:1로 연결됨
- 필수 구성 요소 (인터넷 접속 필요 시)
---

## 라우트테이블 (Route Table)

### 정의
- VPC 내에서 트래픽의 유입, 유출, 이동을 제어하는 규칙 집합
- 각 서브넷은 라우트테이블과 연결되어야 함

### 라우트 (Route)
- **역할**: 라우트테이블과 연결된 서브넷 내에서의 트래픽 유입 및 유출 결정
- **판단 기준**: 대상 주소 IP 프리픽스에 의해서만 결정

### 라우트 설정 필수 요소
| 항목 | 설명 |
|------|------|
| **대상 주소 IP 프리픽스** | 목적지 IP 범위 (예: 0.0.0.0/0) |
| **타겟 리소스** | 트래픽을 전달할 대상 (IGW, NAT, VPN 등) |

### 라우트 우선순위
- 가장 구체적인 경로(longest prefix match)가 우선됨
- 예: 0.0.0.0/0 보다 10.0.0.0/16이 우선
---

## 보안 그룹

### 인바운드 규칙 (Inbound Rules)

**역할**: ENI에 유입되는 트래픽의 허용 여부를 결정

**필수 설정 항목**:
| 항목 | 설명 |
|------|------|
| **소스 (Source)** | 트래픽이 오는 곳 (IP 주소, 보안 그룹 등) |
| **프로토콜** | TCP, UDP, ICMP 등 |
| **포트 범위** | 허용할 포트 번호 (예: 80, 443, 22) |

**목적**: 인바운드 규칙을 추가하여 트래픽이 인스턴스로 유입될 수 있도록 함

### 아웃바운드 규칙 (Outbound Rules)

**역할**: 유출되는 트래픽의 허용 여부를 결정

**필수 설정 항목**:
| 항목 | 설명 |
|------|------|
| **대상 주소 (Destination)** | 트래픽이 가는 곳 (IP 주소 범위) |
| **프로토콜** | TCP, UDP, ICMP 등 |
| **포트 범위** | 허용할 포트 번호 |

### 보안 그룹 (Security Group)
- **스테이트풀(Stateful) 방화벽**: 양방향 트래픽을 자동으로 추적
  - 한 방향으로 트래픽 전달 허용 → 반대방향 응답 트래픽 지능적으로 자동 허용
- **기본 구성**: 모든 VPC에는 기본 보안그룹이 자동으로 생성됨
- **특징**: 인스턴스 레벨에서 트래픽 제어

### 네트워크 접속 제어 목록 (NACL)
- **연결 대상**: 서브넷에 연결되어 해당 서브넷으로 유입/유출되는 트래픽 제어
- **주의사항**: 서브넷 내 인스턴스 간 트래픽 제어 시 NACL 사용 불가 → **보안 그룹 사용**
- **제약**: 하나의 서브넷에는 하나의 NACL만 연결 가능
- **스테이트리스(Stateless)**: 응답 트래픽을 자동으로 허용하지 않음
  - 임시 포트(Ephemeral Port)를 사용하여 명시적으로 아웃바운드 규칙 설정 필요
#### NACL 인바운드/아웃바운드 규칙 구성 요소
| 항목 | 설명 |
|------|------|
| **규칙 번호** | 우선순위 (숫자가 낮을수록 먼저 적용) |
| **프로토콜** | TCP, UDP, ICMP 등 |
| **포트 범위** | 예: 80, 443, 1024-65535 |
| **소스/대상 CIDR** | IP 주소 범위 |
| **동작** | ALLOW 또는 DENY |

## IP 주소
 
### 퍼블릭 IP (Public IP)
- 퍼블릭 인터넷으로 접속 가능한 주소
- **주의**: 인터넷을 통한 연결이 불가능 (자동 할당, 인스턴스 중지 시 회수)

### 탄력적 IP (Elastic IP, EIP)
- 사용자의 요청에 따라 AWS가 계정에 할당하는 퍼블릭 IP 주소
- **특징**: 인스턴스 중지/재시작 후에도 유지되는 정적 IP

### AWS Global Accelerator
- 어디에든 연결할 수 있는 **정적 IPv4 주소** 제공
- 어떤 리전에 있는 리소스라도 서로 연결 가능
- 글로벌 규모의 애플리케이션 최적화
---
 
## NAT 디바이스
 
### 네트워크 주소 변환 (NAT)
- 인터넷 게이트웨이가 퍼블릭 IP 주소를 ENI(Elastic Network Interface)의 프라이빗 IP 주소로 매핑

### NAT의 목적
- 인스턴스가 **인터넷에 접속 가능** ✓
- **인터넷 상의 호스트는 인스턴스에 직접 접속 불가** ✓
- 퍼블릭 IP를 할당하지 않으므로 보안성 향상

### NAT 게이트웨이 (NAT Gateway)
- **EIP 할당**: 탄력적 IP와 함께 생성
- **위치**: 퍼블릭 서브넷에 생성 필수 (인터넷 접속 가능해야 함)
- **특징**: 자동 확장 가능, 관리 간편
- **용도**: 프라이빗 서브넷의 인스턴스가 인터넷에 나가갈 때 사용

### NAT 인스턴스 (NAT Instance)
- **자동 확장**: 불가능 (수동으로 관리해야 함)
- **보안 그룹**: ENI를 지니므로 보안 그룹 적용 필수
- **퍼블릭 IP**: 직접 할당 필요
- **베스천/점프 호스트**: 퍼블릭 IP가 없는 인스턴스에 접속할 때 사용 가능
---
 
## VPC 연결
 
### VPC 피어링 (VPC Peering)
- **연결 대상**: 한 VPC의 인스턴스 ↔ 다른 VPC의 인스턴스
- **제약**: 인스턴스 간 통신만 허용
- **리소스 공유**: 불가능

### AWS Site-to-Site VPN
- **VPG (Virtual Private Gateway)**: 각 VPC마다 별도의 VPG 및 VPN 터널 생성
- **구성**: 여러 VPC 중 하나에 VPN 연결 → 다른 VPC로 트래픽 이동
- **한계**: 다수의 온프레미스 네트워크 연결 시 복잡해짐

### 다중 연결 추천
> **여러 온프레미스 네트워크를 하나의 VPC로 연결할 때**는 **Transit Gateway** 사용 권장
 ---
 
## Transit Gateway
 
### 개요
- **목적**: Direct Connect 링크와 VPN을 사용해 다수의 VPC 및 온프레미스 네트워크 연결
- **특징**: 고가용성 서비스, 중앙 집중식 연결 관리

### Transit Gateway 라우트 테이블
- **역할**: 네트워크 리소스의 트래픽 흐름 제어
- **활용 패턴**:
  - **중앙화 라우터**: 모든 VPC 및 온프레미스 트래픽 제어
  - **격리 라우터**: VPC 간 격리성 유지
  - **공유 서비스**: 서비스 VPC 공유
  - **피어링**: VPC 간 직접 연결
  - **멀티캐스트**: 그룹 통신

### Transit Gateway 피어링
- **리전 간 피어링 가능**: 서로 다른 리전의 Transit Gateway 연결 가능
- **네트워크 확장**: 멀티 리전 아키텍처 구성 용이

### 격리 VPC (Isolated VPC)
- VPC 간 격리성을 유지하려 할 때 유용

### 블랙홀 라우트 (Blackhole Route)
- 특정 라우트 차단 시 Transit Gateway 라우트 테이블에 블랙홀 엔트리 추가
- 트래픽을 의도적으로 폐기
---
 
## AWS Direct Connect
 
### 개요
- AWS 리소스에 대한 **프라이빗, 저지연성 연결** 제공
- **용도**: 대량 데이터 전송, 실시간 데이터 전송

### 호스트 연결 타입
1. **프라이빗 호스트**
2. **공유 호스트**

### 가상 인터페이스 (Virtual Interface)
- **프라이빗 가상 인터페이스**: VPC 내부 리소스 연결
- **퍼블릭 가상 인터페이스**: AWS 공개 서비스(S3, DynamoDB 등) 연결
- **트랜싯 가상 인터페이스**: Transit Gateway를 통한 연결
---
 
## 클러스터
 
### 느슨하게 연결된 클러스터 (Loosely Coupled Cluster)
- 노드 간 통신이 최소화된 구조
- 확장성 우수, 장애 격리 용이

### 긴밀하게 연결된 클러스터 (Tightly Coupled Cluster)
- 노드 간 빈번한 통신이 필요
- 낮은 지연성, 높은 처리량 필요

### 일래스틱 패브릭 어댑터 (Elastic Fabric Adapter, EFA)
- 고성능 컴퓨팅(HPC)용 네트워크 어댑터
- 긴밀하게 연결된 클러스터에서 매우 낮은 지연성 제공
- 과학 계산, 머신러닝 훈련 등에 활용
---
 
## 주요 비교표
 
### 보안 그룹 vs NACL
 
| 항목 | 보안 그룹 | NACL |
|------|---------|------|
| **상태 추적** | 스테이트풀 (O) | 스테이트리스 (X) |
| **연결 대상** | 인스턴스 | 서브넷 |
| **규칙 적용** | 모두 허용/거부 | 규칙 번호 순서대로 |
| **응답 트래픽** | 자동 허용 | 명시적 규칙 필요 |
| **인스턴스 간** | 사용 (O) | 사용 안 함 (X) |
 
### NAT Gateway vs NAT Instance
 
| 항목 | NAT Gateway | NAT Instance |
|------|-----------|-------------|
| **자동 확장** | 가능 | 불가능 |
| **관리** | AWS 관리 | 사용자 관리 |
| **보안 그룹** | 불필요 | 필수 |
| **퍼블릭 IP** | EIP 자동 | 수동 할당 |
| **베스천 호스트** | 불가 | 가능 |
 
### VPC 연결 방식 비교
 
| 방식 | 범위 | 특징 |
|------|------|------|
| **VPC Peering** | 2개 VPC | 간단, 확장성 낮음 |
| **Site-to-Site VPN** | VPC ↔ 온프레미스 | 암호화, 지연 있음 |
| **Transit Gateway** | 다중 VPC + 온프레미스 | 중앙 집중식, 복잡한 연결 지원 |
| **Direct Connect** | VPC ↔ 온프레미스 | 전용선, 고성능, 고비용 |
