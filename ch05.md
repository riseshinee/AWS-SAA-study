## 관계형 데이터베이스

### OLTP vs OLAP

| 구분 | 설명 |
|------|------|
| **OLTP** | 신속한 쿼리 작업에 최적화, 비교적 정형화된 쿼리 사용 |
| **OLAP** | 대규모 데이터에 대한 복잡한 쿼리 작업에 적합, 높은 수준의 컴퓨트 및 스토리지 성능 요구 |

- 스탠다드 데이터베이스 인스턴스
- 메모리 최적화 데이터베이스 인스턴스
- 성능 가속 데이터베이스 인스턴스

### 스토리지 유형

#### 범용 SSD 스토리지
- 최대 **64TB** 볼륨 할당
- 볼륨 당 최대 **16,000 IOPS**
- 최소 스토리지 볼륨 크기: **20GB**
- 초당 가속 성능 계산식:

```
초당 가속 성능 = (크레딧 잔고) / [3000 - 3 × (스토리지 용량 GB)]
```

#### 프로비전 IOPS 스토리지
- 고성능 I/O가 필요한 워크로드에 적합

#### 마그네틱 스토리지
- 레거시 스토리지 타입

## 확장성 (Scalability)

| 방식 | 설명 |
|------|------|
| **수직적 확장** | 애플리케이션/DB는 그대로, 인스턴스 리소스만 추가 |
| **수평적 확장 (스케일아웃)** | 읽기 전용 복제본 추가 |

### 읽기 전용 복제본

- 쿼리(읽기) 작업만 가능한 별도의 DB 인스턴스
- 마스터 데이터는 **비동기적**으로 복제
- 마스터와 다른 리전에 위치 가능
- 생성 가능 수: **RDS 5개 / Aurora 15개**

---

## 고가용성 (Multi-AZ 배포)

- 멀티 AZ 배포 시 **모든 인스턴스는 동일한 리전** 내에 있어야 함
- 하나의 AZ에 **프라이머리(Primary)** 인스턴스 → 읽기/쓰기 담당
- 다른 AZ에 **스탠바이(Standby)** 인스턴스 배치
- 프라이머리 → 스탠바이로 **동기적** 데이터 복제
- 장애 발생 시: RDS가 프라이머리 엔드포인트의 DNS를 스탠바이로 자동 변경

### Aurora 클러스터 유형

| 유형 | 설명 |
|------|------|
| **싱글 마스터** | 프라이머리 인스턴스 + Aurora 레플리카(읽기 복제본) 구성 |
| **멀티 마스터** | 모든 인스턴스가 쓰기 가능, 하나 실패해도 나머지가 공유 클러스터 볼륨에 기록 |

## 백업과 복원

| 용어 | 설명 |
|------|------|
| **RTO** | 데이터 복원 및 처리 재개 가능한 최대 허용 시간 |
| **RPO** | 최대 데이터 손실 허용 기간 |

- RDS는 **EBS 볼륨 스냅샷** 사용
- 스냅샷으로 복구 시 → **새 인스턴스를 생성**한 뒤 데이터 복구

### 자동 스냅샷

- 매일 **30분간의 백업 윈도우** 기간 동안 자동 생성
- 기본 보관 기간: **7일**
- 자동 스냅샷 기능 비활성화 시 → 기존 자동 스냅샷 **즉시 전부 삭제**

- 애플리케이션과 DB 인스턴스를 연결하는 **프록시 서비스**
- 장애 대응 및 재연결 작업을 백엔드에서 자동 처리

## Amazon Redshift

- **데이터 웨어하우스** 서비스 (OLAP에 최적화)
- 복잡한 쿼리를 위해 여러 DB를 하나의 큰 DB로 통합
- **칼럼형(Columnar) 스토리지** → 저장 속도 빠르고 효율적, 개별 칼럼 쿼리 성능 우수

### 데이터 분산 유형

| 유형 | 설명 |
|------|------|
| **EVEN** | 균등 분산 |
| **KEY** | 특정 키 기준 분산 |
| **ALL** | 모든 노드에 전체 복사 |

### Redshift Spectrum

- **S3에 저장된 파일**에서 데이터를 직접 쿼리할 수 있는 서비스

## 비관계형 데이터베이스 (NoSQL)

- 비구조화 데이터 저장에 최적화
- **Key-Value** 구조, 스키마 없음
- 기본 키 기준 쿼리에 최적화

## Amazon DynamoDB

### 기본 개념

| 용어 | 설명 |
|------|------|
| **파티션** | 테이블을 위한 스토리지 할당 영역 |
| **파티션 키** | 데이터 분산의 기준 키 |
| **소트 키** | 동일 파티션 내 정렬 기준 키 |
| **속성** | 하나의 key-value 쌍 |
| **아이템** | 하나 이상의 속성의 집합 |

### 읽기 일관성

- **강한 일관성의 읽기 (Strongly Consistent Read)**: 최신 데이터 반환 보장
- **종국적 일관성의 읽기 (Eventually Consistent Read)**: 성능 우선, 약간의 지연 허용

### 조회 방법

| 방법 | 설명 |
|------|------|
| **스캔 (Scan)** | 테이블 내 모든 아이템 반환 |
| **쿼리 (Query)** | 파티션 키 값에 따라 아이템 반환 |

### 보조 인덱스

- **전역 보조 인덱스 (GSI, Global Secondary Index)**: 파티션 키와 소트 키 모두 변경 가능
- **지역 보조 인덱스 (LSI, Local Secondary Index)**: 파티션 키는 동일, 소트 키만 변경

### 기타 기능

- **오토 스케일링**: 트래픽에 따라 처리 용량 자동 조정
- **전역 테이블 (Global Tables)**: 멀티 리전 복제
- **백업**: 온디맨드 백업 및 특정 시점 복구(PITR) 지원
